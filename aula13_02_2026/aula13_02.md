<h1> Criando Testes Unitários com Unittest</h1>

<h3>Exercício 1: O Verificador de Idade</h3>

<u>Objetivo: </u> 
&nbsp;
Praticar a estrutura básica e assertEqual.
• Tarefa: Crie uma função pode_dirigir(idade) que retorna True se a idade for
maior ou igual a 18, e False caso contrário.
• Desafio: Escreva uma classe de teste com dois métodos: um para testar um caso
positivo (ex: 20 anos) e outro para um caso negativo (ex: 16 anos).

<h3>Exercício 2: Calculadora de Descontos</h3>
<u>Objetivo: </u>

&nbsp;
 Praticar lógica de negócio e múltiplos asserts.
• Tarefa: Crie uma função calcular_desconto(valor, percentual).
‣ Se o percentual for maior que 50%, a função deve limitar o desconto a 50%
(regra de negócio).
• Desafio: Crie testes para:
‣ Um desconto comum (ex: 10% de 100).
‣ O limite de segurança (ex: tentar aplicar 70% de desconto e verificar se o
resultado reflete apenas 50%).
‣ Um valor de produto zero.

<h3>Exercício 3: Sistema de Cadastro de Usuários</h3>
<u>Objetivo: </u>

&nbsp;
Testar validações de regras de negócio usando assertRaises.
• Tarefa: Crie uma função cadastrar_senha(senha).
‣ A senha deve ter pelo menos 8 caracteres.
‣ Se a senha for menor que isso, a função deve levantar um ValueError com a
mensagem “Senha muito curta”.
• Desafio:
‣ Teste de Sucesso: Verifique se uma senha de 10 caracteres não levanta erro (ou
simplesmente passa).
‣ Teste de Exceção: Use with self.assertRaises(ValueError): para garantir
que tentar cadastrar a senha “123” interrompa o processo corretamente.

<h3>Exercício 4:Validador de Senha Avançado</h3>
<u>Objetivo: </u>

&nbsp;
Praticar assertTrue, assertFalse e validação de múltiplas condições.
• Tarefa: Crie uma função validar_senha_avancada(senha) que retorna True se a
senha atender aos seguintes critérios:
‣ Pelo menos 8 caracteres.
‣ Conter pelo menos um número.
‣ Conter pelo menos uma letra maiúscula.

Retorna False caso contrário.
• Desafio:
‣ Escreva testes para:
– Uma senha válida (ex: “Senha123”).
– Uma senha muito curta (ex: “Curta1”).
– Uma senha sem número (ex: “SemNumero”).
– Uma senha sem letra maiúscula (ex: “semliteramaiuscula1”).

<h3>Exercício 5: Validador de CPF</h3>
<u>Objetivo: </u>

&nbsp;
Praticar a escrita de casos de teste simples e o uso de asserções booleanas.
Imagine que você está desenvolvendo um sistema de cadastro e precisa de uma
função que verifique apenas se o formato básico de um CPF (sem pontos ou traços)
é válido antes de processar os cálculos matemáticos.
• Tarefa: Crie uma função validar_formato_cpf(cpf) que:
• Retorna True se:
‣ For uma string.
‣ Tiver exatamente 11 caracteres.
‣ Todos os caracteres forem dígitos numéricos.
• Retorna False
‣ em qualquer outro caso (comprimento errado, caracteres alfabéticos ou tipo de
dado incorreto como int).
• Desafio de Teste: Escreva uma classe de teste utilizando o módulo unittest que
verifique:
‣ Sucesso: Um CPF válido (ex: "12345678901").
‣ Falha por caracteres: Um CPF com letras (ex: "1234567890A").
‣ Falha por tamanho curto: Um CPF com 10 dígitos (ex: "1234567890").
‣ Falha por tamanho longo: Um CPF com 12 dígitos (ex: "123456789012").
‣ Falha por tipo: Uma entrada que seja um número inteiro em vez de string (ex:
12345678901).
